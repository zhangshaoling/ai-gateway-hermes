# AI Gateway Hermes

统一 AI 服务接入网关 — 多 Provider 智能路由与负载均衡。

### 设计原理

AI 服务的**多样性**和**不可靠性**是这个网关要解决的两个核心问题：

- **多样性** — 十几个 Provider、上百个模型，需要一个统一的接入层
- **不可靠性** — 单一 Provider 随时可能故障、限流、降质，必须自动切换

### 核心能力

| 能力 | 原理 |
|------|------|
| **智能路由** | 按延迟/成本/成功率动态分配请求，不把鸡蛋放一个篮子 |
| **故障转移** | 请求失败后毫秒级切换到备用 Provider，用户无感 |
| **密钥管理** | 多 API Key 轮转，单 Key 超限自动跳过 |
| **统一接口** | OpenAI 兼容格式，换 Provider 不改代码 |

### 技术栈

Python · FastAPI · 异步非阻塞 · LRU 缓存
